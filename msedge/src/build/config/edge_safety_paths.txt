# Edge Safety Checker — paths control file.
# Controls which directories/files are checked by the edge-safety clang plugin.
#
# Format:
#   # comment
#   $ relative/path/to/source/root  (from this file's directory)
#   . group1,group2,...             (check groups to enable; omit for all)
#   + path/to/include/              (opt-in this directory or file — prefix)
#   - path/to/exclude/              (opt-out this directory or file — prefix)
#   ~-third_party/                  (opt-out any path containing pattern)
#   ~+first_party/                  (opt-in any path containing pattern)
#   @ reserved for future use
#
# Check groups: lifetime, ipc, cpp, memory, integer, url, design,
#               injection, sensitive, webui, deser, value, all
#
# Matching order:
#   1. Longest prefix match (+/-) is checked first.
#   2. If no prefix matches, substring rules (~+/~-) are checked.
#      If multiple substring rules match, the longest pattern wins.
#   3. If nothing matches, the file is checked by default.
#   4. #pragma allow_edge_safety in a file overrides everything.
#
# Policy: Edge-only scanning.
#   The edge-safety plugin is intended to enforce the Edge secure-coding guide
#   on code authored by Edge developers (Microsoft downstream). Upstream
#   Chromium code should not be scanned. Edge-authored code lives under paths
#   containing "edge" (e.g. chrome/browser/edge_copilot/, edge_webui/,
#   third_party/blink/renderer/edge_*/, etc.) per Edge naming convention.
#
#   Strategy: substring layer is used as a default-deny gate. The "~-/" rule
#   matches every path (every relative path contains a '/'); the "~+edge" /
#   "~+Edge" rules opt back in any path that contains the substring.
#   Longer-pattern-wins resolution makes the opt-ins override the deny for
#   Edge paths.

# Enable all check groups.
.all

# Path from this file to the source root.
$../../

# --- Build tooling / generated output (always skipped) ---
# These take prefix priority over the substring layer below, so even
# Edge-authored files that land here (e.g. tools/clang/plugins/EdgeSafety*.cpp)
# are not self-scanned.
-out/
-build/
-buildtools/
-tools/
-testing/

# --- Default deny (substring layer) ---
# Every path contains '/' once stripped to source root; this serves as the
# catch-all deny for upstream Chromium files that lack an "edge" component.
~-/

# --- Opt-in: any path containing "edge" / "Edge" ---
# Matched by Edge-authored code: edge_copilot/, edge_webui/, edge_safety/,
# EdgeFoo.cpp, etc. Longer than "~-/" so it wins over the default deny.
~+edge
~+Edge

# Nocompile tests (.nc) use -Xclang -verify which requires every diagnostic
# to be annotated. Edge-safety warnings are unexpected by these files.
# Longer than "~+edge" / "~+Edge" so it wins for any Edge-authored .nc files.
~-_nocompile.nc

# --- Per-file opt-outs (Edge-authored files with intentional violations) ---
# Prefix rules win over substring rules, so listing a file here suppresses the
# edge-safety plugin even though its path contains "edge".
#
# Each entry should include a one-line justification. Prefer fixing the
# underlying issue when feasible; opt-outs are an escape hatch for code that
# legitimately needs the banned construct (e.g. COM interop requiring
# try/catch around native calls that throw _com_error / structured exceptions).

# Wraps native COM/Win32 text-services calls (translation, speech, OCR) that
# can throw structured exceptions; the 21 try/catch sites are intentional.
-chrome/services/edge_text_services/edge_text_services_manager_impl.cc
